R & RStudio

Stata is used for statistics in health economics at universities a lot, in particular for cross section and panel analysis. But it was not very well suited for solving standard quadratic programming problems. Also, we found out that the Stata community was not very supportive (to put it mildly) when discussing how to code a non standard quadratic programming problem. This is were R came to the rescue. It was the starting point for developing a preference for coding in R.

Read more 2015

Git & Github

Collaboration when writing code is a big thing. Not only would one use versioning to keep track of the older versions of your files. When working in a project team, one also would like to work on code files at the same time without loosing track of who did what when all the time. Although I used Subversion as a version control system on my NAS previously, I switched to Git immediately when reading online forums on Git and the Git manual. Embracing Github followed along the way.

Read more 2016

Machine learning

Standard linear regression is great for solving lots of prediction problems, even those for which better estimation techniques apply theoretically. Machine learning on the other hand produces even better preditions, although they come at the risk of overfitting your models. A combination of the worlds of statistics and machine learning would be optimal. This I found in the Stanford online course called "Statistical machine learning", which I highly recommend. Code in R, of course.

Read more 2016

Microsoft Azure ML & API output

Now it was time to find out how to create my own web apps such that they can be published remotely or locally. The Microsoft Azure ML suite offers these kind of facilities out of the box. Major advantage: you can write your own code in R and combine it with the out of the box tools. But most of all: an API is automatically created after estimation of your model and a demo how to call this API externally (even from good old Excel) is also provided.
